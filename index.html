    Etiquetas

    El código en HTML está formado por un conjunto de etiquetas (tags).
    El navegador interpreta estas etiquetas al momento de cargar la página.

    Simbolos:  

    <>

    Estrutura Basica:  

    <Nombre de la etiqueta>


    La mayoría de las etiquetas vienen en pares, es decir:
    Etiqueta Apertura y Etiqueta Cierre (contienen una diagonal /)

    De esta manera: 

    <EtiquetaApertura>Contenido<EtiquetaCierre/>

    Y otras que solo tienen la etiqueta de apertura

    Ejemplos: 

     <img> 
 <input>
 <br>

    Elementos

    Las etiquetas junto con su contenido, forman un elemento HTML.

    Ejemplo: 

    <h3>Ejemplo de elemento<h3>

    Tipos de elementos

    Todos los elemetos se pueden dividir en dos tipos basicos:En linea y en bloque

    En Bloque
    Un elemento en bloque es aquel que contiene un bloque significativo de contenido que debería mostrarse en su propia línea.Por ejemplo: párrafos, tablas, listas

    En Linea
    Los elementos en línea usualmente contienen menos texto y se muestra junto a otro texto en la misma línea, cómo cuando se enfatizan palabras en una oración

    Atributos

    La etiqueta de apertura de un elemento puede uno o varios atributos que proporcionan más información sobre el elemento

    Ejemplo: 

    <p class=“mensaje”> Hola Mundo <p/>

    En este caso [class="mesaje"] es el Atributo

    Atributos estandar

    Este tipo de atributos son los siguientes


    1.-class

    Indica la clase a la cual un elemento pertenece. Cualquier número de elementos pueden pertenecer a una clase. Un solo elemento puede pertenecer a muchas clases


    2.-style

    Especifica propiedades CSS de un elemento


    3.-id

    Especifica un identificador único para un elemento

    Comentarios

    Los comentarios en HTML son notas que no se muestran en el navegador y que se pueden leer revisando el código fuente

    Ejemplo:

    El Head

    El elemento head contiene información sobre el documento HTML, incluyendo palabras clave, enlaces a otros documentos que usa como archivos CSS y JavaScript
    A excepción del elemento "title" que se muestra en la barra de título del navegador, ningún otr elemento se muestra en el contenido de la página HTML

    Ejemplo:


    Los elementos de head son los siguietes

    <title>:

    Permite agregar un título a un documento HTML, se muestran en la barra de título de los navegadores y se usa como valor por default cuando se marca una página como favorita.


    <link>:

    define las relaciones entre dos documentos. Su uso más frecuente es para referirse a las hojas de estilo que usa el documento.
    Ejemplo:
    <link rel="stylesheet" type="text/css" href="main.css" media="screen" />


    <meta>:

    El término meta se refiere a metadatos. Su uso más común es para describir información sobre la descripción y palabras clave del documento, y el tipo de codificación.


    <meta>:

    Permite integrar lenguajes de scripting en un documento HTML.
    Ejemplo
    <script type= “text/javascript” src=“js/miscript.js”></script>

    El Body

    Todo el contenido de un documento HTML se encuentra en el elemento body.
    El elemento body va después del elemento head y debe cerrarse antes de la etiqueta de cierre del elemento html.

    Selectores

    Definen que elementos HTML serán afectados por las declaraciones
    Existen varios tipos de selectores
    Elementos
    Clases
    ID’s
    Descendientes
    Hijos
    Adyacentes (hermanos o siblings), etc.

    Selector Universal

    El selector universal, selecciona todos los elementos de un documento HTML.
    Se denota por un “*”

    Selector tipo Etiqueta

    Selectores tipo etiqueta o de elementos, son los más simples y comunes.

    Selector de Clase

    Se utiliza para especificar un estilo a un grupo de elementos con la misma clase.
    Su principal característica, es que se identifica por un punto.
    Ejemplo: 

    <h1 class=”alerta”> Hola </h1>

    Selector de ID

    Se utiliza para especificar un estilo a un elemento mediante su id.
    Su principal característica es que se identifica por un sharp o gato “#”
    Ejemplo: 

    <id=”contenido”> Esto es un div </div>

    Diferencia entre Clase e ID

    Clase: se utiliza para un grupo de elementos, es decir como el apellido de nuestros elementos.
    ID: se utiliza para un "único elemento", no se repite ese id, es decir, como un numero de INE o CURP de nuestro elemento.

    Selector Descendientes

    Afectan a elementos que están contenidos dentro de otro(s) elementos.
    Lo usamos mediante el uso de un espacio entre cada elemento.

    Selector de Hijo

    Similares a los selectores descendientes. Pero solo afectaran a los hijos directos, es decir a los descendientes directos.
    Lo usamos con el símbolo “>”

    Selector de Hermano

    Los selectores de hermanos se emplean para seleccionar elementos adyacentes, es decir que su elementos padre es el mismo, y ambos elementos están al mismo nivel.
    Hay dos tipos de selectores hermanos:
    Selector Directo
    Selector General

    Selector de Hermano Directo

    Se utiliza cuando el hermano esta inmediatamente después de él.
    Lo usamos mediante el símbolo “+”
    A + B = El elemento B debe de estar inmediatamente después del elemento A en el código HTML.

    Selector de Hermano General

    Se utiliza cuando el hermano esta después de él. Lo usamos con el símbolo “~”
    A ~ B = El elemento B debe de estar después del elemento A en el código HTML.

    Agrupación de Selectores

    La agrupación de selectores se utiliza cuando uno o más selectores tienen las mismas propiedades.
    Esto nos ayudará a ahorrar líneas de código. Agrupamos con el uso de la coma “,”

    Pseudo-Clases

    :activeLos ‘links’ activos.
    :hoverEl elemento en el momento que pasa el mouse sobre él.

    :not

    :not(x)Selecciona todos los elementos que no corresponden al elemento X Es decir que permite excluir al elemento X.

    Prefijos

    En CSS3 existen propiedades nuevas que cada navegador interpreta de forma diferente.
    Al no haber un estándar por el que se tengan que regir los navegadores, se deben usar los llamados “prefijos”.

    -webkitPara:Chrome,Safari , Konqueror, etc…
    -mozPara:Firefox
    -oPara:Opera
    -msPara:Internet,Explorer

    Propiedad Gradient

    Hay dos tipos principales de Gradientes:
    Linear
    Radial
    Debemos tomar en cuenta que esta propiedad requiere prefijos. 

    selector {
background: -prefijo-linear-gradient (dirección, color1,  color2, …);
}

    
div {
background: -webkit-linear-gradient (top,red,blue);
}
    direcciónDesde donde empezará el Gradiente y puede ser:
    Grados (90deg, 5deg).
    Lados definidos (top, right, bottom,left).
    color#El primer color será el cual empiece el gradiente, el segundo, el tercero, etc… irán tomando sus posiciones respectivamente.

    Propiedad Transform

    Esta propiedad aplica una transformación a un elemento en 2D o 3D.
    Permite: rotar, escalar, mover, sesgar, etc. 

    selector {
-prefijo-transform: método (valores);
}
    translate()Mueve al elemento desde su posición original en función de los parámetros (x,y)
    rotate()El elemento gira en sentido de las manecillas del reloj en un grado determinado, las unidades son: “deg”.
    scale()/th>El elemento aumenta o disminuye el tamaño dependiendo de los parámetros (x,y).
    skew()El elemento se sesga en un ángulo según los parámetros (x,y).
    matrix()Este método toma 6 parámetros que le permiten, rotar, escalar, trasladar, y sesgar.
    -originPor defecto en una transformación el origen (0,0) toma el centro del objeto, esta propiedad permite mover el punto de origen.
    rotateX()El elemento gira sobre el eje X
    rotateY()El elemento gira sobre el eje Y
    rotate3(XYZ)Define una rotación similando un efecto 3D.
    skewX()Define una transformación de sesgo en el eje X

    Propiedad Transitio

    Esta propiedad nos brinda una manera muy sencilla de animar los cambios en las propiedades CSS, en lugar de que los cambios se produzcan de manera instantánea.

    selector {
-prefijo-transition: [Propiedad CSS]  [Tiempo] [Función de Tiempo] [Retardo]
}
    Propiedad CSSSe refiere a la propiedad CSS que será animada.
    Generalmente se coloca el valor “all”.
    TiempoSe refiere al tiempo que durará la transición, puede ser en segundos o en milisegundos.
    Func. TiempoSe refiere a funciones predefinidas, que permiten variar el avance de la transición.
    RetardoSe refiere al tiempo que tardará la transición en empezar

    Propiedades CSS que se pueden transicionar

    • background
    • border
    • box-shadow
    • color
    • font-size
    • font-weight
    • opacity
    • z-index
    • transform
    • line-height
    Funciones de tiempo para transicionar.
    linearLa transición se realiza de manera uniforme.
    easeAcelera al inicio, se retarda y acelera al final.
    ease-inSe retarda al inicio, pero se acelera al final.
    ease-outAcelera al inicio, pero se retarda al final.
    ease-in-outAcelera al inicio, se acelera y se retarda al final.

    Animaciones

    En CSS3 podemos animar la transición entre un estilo y otro. Las animaciones constan de dos componentes:
    Un estilo que describe la animación.
    Un conjunto de ‘fotogramas’ que indican su estado inicial y su estado final

    animationNombre de la propiedad CSS3
    @keyframesPermite declarar los fotogramas claves.

    Propiedad Animation

    Esta propiedad consta de varias sub-propiedades.
    Con ellas podemos configurar el ritmo, la duración, el nombre, la función de tiempo y otros detalles sobre la secuencia de la animación.
    Ejemplo de una regla animation: 

    selector { 
-prefijo-animation: 3s normal 1 mianimacion;
}

    CSS3 nos brinda una sintaxis abreviada de esta propiedad:
    animation: duration timing-function delay direction iteration-count name;

    animation-delayTiempo de retardo en el comienzo de la animación.
    animation-directionIndica el comportamiento de la animación al inicio y al final.
    animation-durationCantidad de tiempo que tarda en completar su ciclo la animación.
    animation-iteration-countNúmero de veces que se repite
    animation-nameNombre de la animación.
    animation-fill-modeQué estilos se aplicarán al elemento cuando la animación no se este ejecutando.
    animation-play-statePermite pausar y reanudar la animación
    animation-timing-functionIndica el ritmo de la animación
    Sub-Propiedad (animation-direction)
    normalLa animación se reinicia comenzando desde el principio.
    alternateLa animación, al terminar un ciclo, invierte su dirección.
    reverseCada ciclo de la animación se reproduce al revés.
    alternate-reverseEs una combinación de las dos anteriores
    Sub-Propiedad (animation-timing-function)
    linearLa animación se realiza de manera uniforme
    easeAcelera al inicio, se retarda y acelera al final.
    ease-inSe retarda al inicio, pero se acelera al final
    ease-outAcelera al inicio, pero se retarda al final.
    ease-in-outAcelera al inicio, se acelera y se retarda al final
    steps(num,end)Divide la animación en un numero de pasos.

    Propiedad @keyframes

    Permite controlar los pasos en una secuencia de animación CSS, mediante el establecimiento de puntos de trayectoria.
    Para usar esta propiedad se crea una regla @keyframes con el mismo nombre declarado en la propiedad animation-name.
    Ejemplo de una regla @keyframes 

    @-prefijo-keyframes mianimacion {  
       from  { propiedad: valor }
       to  { propiedad: valor }
}
    fromEstado inicial de la animación
    toEstado final de la animación.
    0%Estado inicial de la animación.
    100%Estado final de la animación.
    x%Estado de la animación en un porcentaje x de tiempo.

    Propiedad @keyframes y animation

    Ambas propiedades trabajan de la mano, es decir una no puede trabajar si la otra no existe.
    Es importante saber que estas dos propiedades requieren del uso de prefijos.

    Diseño responsivo

    ¿Qué es?

    El diseño responsivo es un diseño que responde al tamaño del dispositivo desde el que se está visualizando la web, adaptando las dimensiones del contenido y mostrando los elementos de una forma ordenada y optimizada.
    Consta de una serie de reglas de diseño web que permiten al contenido de un sitio web adaptarse a diferentes resoluciones en diferentes dispositivos.
    En el diseño responsivo haremos mucho uso de medidas de porcentaje o medidas relativas tales como “em” , “%”, etc…
    Aun así es válido poder trabajar con pixeles.

    Ventajas

    Reducción de costos de producción
    Eficiencia en el mantenimiento
    Mejor usabilidad
    Impacto en el Visitante

    Media Queries

    ¿Qué son?

    Permite la representación del contenido para adaptarse a la resolución de la pantalla. Añadido en CSS3 permiten que el contenido se adapte a un rango específico sin tener que cambiar el contenido.

    Condiciones

    ¿Qué puede ser una condición? Usamos el diseño responsivo porque el ancho de los dispositivos varían, debemos de considerar que “la condición debe de ir de acuerdo al ancho de pantalla”

    Sintaxis

    @media not|only mediatype and (media feature) {    
CSS-Code;
}

    Variables

    Una variable puede tener un nombre corto (como xey) o un nombre más descriptivo (edad, carname, total_volume).
    Reglas para las variables de PHP:
    Una variable comienza con el signo $, seguido del nombre de la variable
    Un nombre de variable debe comenzar con una letra o el carácter de subrayado
    Un nombre de variable no puede comenzar con un número
    Un nombre de variable solo puede contener caracteres alfanuméricos y guiones bajos (Az, 0-9 y _)
    Los nombres de las variables distinguen entre mayúsculas y minúsculas ($ age y $ AGE son dos variables diferentes)
